## 2. 작업 규칙

### 배치
- 새 항목은 [README.md](./README.md)의 "배치 규칙"을 따른다.
- **카테고리 판단**: 이 맥에서 실가동되는 시스템(자체 MCP·스크립트 등) → `라-custom-system`, 반복업무 → `나-routine`, 생애주기(흐름) 있는 작업 → `가-process`, 애매 → `다-parking`.
- **도메인 판단**: 개발·연구·지식 → `1-research`, 회사운영 → `2-business`, 사적 → `3-personal`. (`다`·`라`는 도메인 없음)
- **Process 단계 판단**: 참고용→reference, 아이디어→inbox, 진행→action, 완료→result, 배포·보관→share. (inbox는 new/renewal 구분 없이 프로젝트 직속)

### 넘버링
- **넘버링 체계의 정본은 [README.md](./README.md)** — 카테고리·도메인·단계 코드 표를 그대로 따른다.
  (요약 예: 가-process `10`=research·reference · `31`=personal·inbox · 나-routine `10`=research·routines)
- 작업 원칙: 새 시스템(코드) 폴더는 기존 패턴을 잇고, **코드 폴더는 임의로 변경하지 않는다.**

### 날짜 프리픽스 — 프로젝트/콘텐츠 폴더에 적용 (핵심 규칙)
- **프로젝트·콘텐츠·결과 폴더를 만들 땐 날짜 프리픽스 `YYYY_MM_DD_` 를 붙인다.**
  - 예: `2026_03_30_work-journal`, `2026_05_07_탄넨바움 단원음악회`, `2026_06_09_remotion_media`
  - 효과: 폴더 목록이 **생성일 순으로 자동 정렬**된다.
- **사용자가 날짜를 명기하지 않고 (프로젝트/콘텐츠) 폴더 생성을 지시하면** → 만들기 전에
  **"생성일(오늘)로 날짜 붙여 만들까요?"** 라고 한 번 확인한다. 기본값은 **"날짜 붙여 생성"**.
- **예외 (날짜 안 붙임)**: ⓐ 시스템 코드폴더(카테고리·도메인·단계·루틴버킷 — 번호로 식별) ⓑ 루틴 정의폴더(`routines/<루틴명>` — 기능명) ⓒ `라-custom-system` 하위 시스템 폴더(시스템명).
- **날짜 = 최초 생성일** (기준 우선순위):
  1. git 저장소면 **첫 커밋일** (`git log --reverse --date=format:%Y_%m_%d --format=%cd | head -1`) — 가장 정확
  2. 그 외 폴더는 **폴더 생성시각**(`stat -f %SB -t %Y_%m_%d`)
  3. 미디어 등 의미 있는 원본일이 따로 있으면 그 날짜 (예: 영상 원본 수정일) — 사용자 판단
- 미디어 묶음은 폴더로 만들어 같은 규칙 적용 (예: `2026_04_27_탄넨바움 단원음악회/` 안에 영상들).
- **단독 파일**(zip·pdf·pptx·mp4 등 폴더에 안 든 것)은 자동 적용하지 않는다 → 사용자가 개별 지시.
- 이미 `YYYY_MM_DD_`로 시작하면 다시 붙이지 않는다(중복 방지).

### 삭제·이동 안전
- 삭제·이동 전 **대상 내용을 먼저 확인**한다.
- **git 저장소가 아닌 폴더**는 삭제 시 영구 소실 → 반드시 사용자 확인.
- git 폴더는 원격 푸시(백업) 여부를 확인한 뒤 삭제.
- 빈 분류함을 미리 만들지 않는다 (내용이 생길 때 폴더 생성).

### 단계 간 이동 시 git 처리 (필수 문답)
- 프로세스 진행으로 프로젝트를 **시스템 단계 간 이동**할 때(예: `inbox → action`, `action → result`), 그 폴더에 **`.git`이 있으면 이동 전 반드시 묻는다**:
  - ⓐ **그대로 이동(git 유지)** — 폴더째 `mv`. 저장소가 따라가고 **위치(참조)만 바뀐다**(이력 보존).
  - ⓑ **git 제거 후 이동** — 이전 단계의 `.git`을 지우고 이동(이력 불필요·새 출발).
- **묻지 않고 임의 처리하지 않는다.** 이유: 단계마다 `.git` 등 **숨김 폴더가 과적재**되어 관리불능이 되는 것을 막기 위함.
- 원격(remote) 연결이 있는 ⓐ의 경우, 원격은 그대로 유효(경로 무관)함을 사용자에게 알린다.

### 구조 인덱스 자동 갱신
- `STRUCTURE.md`는 **폴더만, 날짜 프로젝트 폴더(`YYYY_MM_DD_…`) 깊이까지** 추적하는 자동 인덱스다.
  - 프로젝트 폴더 *내부*(하위 폴더·파일)는 추적하지 않는다 → 필요하면 그 폴더를 직접 `ls`.
  - **파일은 인덱스에 절대 나열하지 않는다** (단독 파일은 "든 폴더"까지만 보인다).
- **Claude Code**: 훅(`.claude/hooks/on-bash.sh`)이 구조변경 명령(mkdir/mv/cp/rm) 뒤 `update-structure.sh`를 자동 실행한다. 손 안 대도 갱신됨.
- **Antigravity·Codex 등 다른 엔진**: 훅이 없으므로, 폴더를 **추가·삭제·이동·이름변경한 직후 `bash update-structure.sh`를 직접 실행**한다.
- `STRUCTURE.md`는 **직접 편집 금지** — 항상 스크립트로만 갱신(드리프트 방지). 스크립트는 폴더 구조가 실제로 바뀐 경우에만 다시 쓴다.

### 단독 파일 배치 — 구조폴더 바로 밑은 폴더로 감싸기 (강제)
- **구조(코드)폴더**(= 날짜폴더가 아닌 시스템 폴더: 카테고리 `가/나/다/라`, 도메인 `1~3`, 단계 `10~34`, 루틴 버킷 `10/11` 등, **다-parking·라-custom-system 포함**) **바로 밑에 단독 파일을 두지 않는다.**
- 구조폴더 바로 밑에 파일이 놓이면(zip·pdf·pptx 등 **종류 불문**), **반드시 `YYYY_MM_DD_파일명`(확장자 제외) 폴더를 만들어 그 안에 넣는다.** 날짜 = **파일 생성일(birth)** = `stat -f %SB -t %Y_%m_%d`.
- **예외**: **날짜 프로젝트 폴더(`YYYY_MM_DD_…`)·루틴 폴더(`<루틴명>`) 안의 파일**은 단독으로 둬도 된다. 건드리지 않는다.
- 새 파일을 구조폴더 바로 밑에 두라는 지시를 받으면, 위 규칙대로 폴더로 감싸 넣고 그 사실을 알린다.

## 3. 파일찾기 (File Find) — 필수 동작

사용자가 무언가를 "찾아줘 / 어디 있어 / where is / find" 등으로 요청하면, 엔진(Claude·Codex·Antigravity)과 무관하게 **반드시 아래 절차를 따른다.**

1. **범위 확인 (검색 실행 전 필수)** — 찾기를 돌리기 전에 **두 축**을 물어 범위를 좁힌다. 둘 다 사용자가 이미 명시한 축은 생략한다.
   - ① **대상**: 폴더 / 파일 / 둘 다
   - ② **매칭 방식**: 의미(동의어·연관개념) / 키워드(이름 문자열) / 둘 다
   - 답에 따라 검색 종류(`find -type d` / `-type f` / 둘 다)와 매칭 폭을 조정해 실행한다. → 노이즈 감소·정확도 향상.
2. **검색 실행** — 위 범위에 맞춰 찾는다. **먼저 [STRUCTURE.md](./STRUCTURE.md)로 후보 범위를 좁히고**, 전체 `find`/`ls`는 피한다(탐색 토큰 절약). 인덱스로 좁혀지지 않을 때만 해당 단계 폴더를 직접 탐색한다.
   - **키워드 모드**: 파일/폴더 *이름* 문자열 일치 (한/영·약어·날짜 프리픽스 `YYYY_MM_DD_` 고려).
   - **의미 모드**: 동의어·연관개념까지 확장 (예: "음악회" → `단원음악회`·`정기연주회`·`탄넨바움`).
   - SDK·`node_modules`·`.git`·`.venv` 등 노이즈 경로는 제외한다.
3. **후보를 링크로 제시** — 찾은 폴더·파일 경로를 **클릭 가능한 마크다운 링크**로 제공해, 사용자가 링크로 해당 위치에 바로 이동할 수 있게 한다.
   - 형식: `[표시이름](상대경로)` — 폴더는 폴더 경로, 파일은 파일 경로.
   - 예: `[실내건축기술사 폴더](가-process/3-personal/32-action/2026_05_27_실내건축기술사 자격신설추진/)`
   - 경로는 워크스페이스 루트(`~/work`) 기준 상대경로로 적는다.
4. **관련도 순 정렬** — 후보가 여럿이면 관련도 높은 순으로 제시하고, 각 후보가 왜 매칭됐는지 한 줄로 덧붙인다.
5. 결과가 없으면 추측해 단정하지 말고, 검색 범위·키워드를 좁히거나 넓혀 다시 물어본다.

> 링크 형식은 모델 중립이다. 마크다운 상대경로 링크는 IDE·터미널·각 엔진 UI에서 공통으로 클릭 이동을 지원한다.

## 4. Routine 푸시 프로토콜 — 필수 동작

`나-routine/<도메인>/`은 **routines**(작업공간·양식)와 **outputs**(결과 누적)로 이뤄진다.
사용자가 특정 루틴에 대해 **"푸시해 / 아웃풋해 / 결과 이동해 / push"** 라고 하면, 엔진 무관하게 **반드시**:

1. **결과 이동** — `routines/<루틴명>/`의 정제된 결과를 `outputs/<루틴명>/`로 보낸다.
   대상 폴더가 없으면 만들고, 결과는 `YYYY_MM_DD_…`(오늘) 폴더에 넣어 시점을 기록한다.
2. **작업공간 정리** — 그 루틴의 설정(`routines/<루틴명>/_routine.md`의 `after_push: empty|keep`)대로:
   - `keep` → 양식·템플릿만 남기고 채워진 작업 데이터만 비운다 (예: 업무일지).
   - `empty` → 작업공간을 비운다 (예: 뉴스수집).
   - 설정이 없으면 **사용자에게 비움/유지를 한 번 묻고**, 답을 `_routine.md`에 적어 둔다.
3. **출력 형태**는 루틴 설계 따름 — 데이터만(예: SQLite) 또는 채워진 양식 통째. 시스템은 강제하지 않는다.
4. 루틴 폴더(`<루틴명>`)는 지속 반복이라 **날짜 프리픽스를 붙이지 않는다**(기능명 그대로). 결과(outputs)만 날짜를 붙인다.

### `_routine.md` 양식 (각 루틴 폴더에 하나)
새 루틴을 만들 때 `routines/<루틴명>/_routine.md`를 아래 형식으로 둔다.
```markdown
---
routine: 업무일지
after_push: keep        # keep(양식 유지) | empty(작업공간 비움)
output_format: filled   # filled(채워진 양식 통째) | data(데이터만, 예 SQLite)
---
매일 업무 내용을 양식에 기록한다. "푸시해" 하면 그날 기록을 outputs로 보내고 양식은 남긴다.
```
예2) 뉴스수집: `after_push: empty`, `output_format: filled` — 정제 결과만 outputs로, 작업공간은 비움.

## 5. 비즈니스 폴더 생성 (수주·계약 프로젝트) — 엔진 공통 절차
회사 **수주·계약(발주자 있는)** 프로젝트 폴더는 **`가-process/2-business/` 하위에만** 만든다(신규=`21-inbox`, 진행=`22-action`). 아래를 문답으로 진행한다.
- **스킬 지원 엔진(Claude Code·Codex 등)**: 동일 절차가 `business-folder-creator` 스킬로 제공된다. 원본은 `.agents/skills/business-folder-creator`(중립), `.claude/skills/`는 그 심링크 → 단일 원본.
- **스킬 미지원 엔진(Antigravity 등)**: 이 §5 절차를 평문으로 직접 수행한다.
- **이 §과 스킬은 같은 절차** — 한쪽을 고치면 다른 쪽도 맞춘다(드리프트 방지).

1. **폴더명** = `YYYY_MM_DD_발주자_계약명_생성자` (필드 내부 공백→`-`, 구분→`_`). 발주자·계약명·생성자·날짜(기본 오늘)·대상 단계를 묻는다.
2. **git**: 관리 여부 + 위치(로컬 `git init` / GitHub `gh repo create <name> --private|--public --source=. --remote=origin` / 기타 원격 URL) 문답 후 구축.
3. **커밋 태그 분류**(도면·문서·엑셀·사진/현장·발주자제공자료·계약/공문 등)를 물어 폴더 루트 `_project.md`의 `commit_tags`에 기록.
4. **커밋 헬퍼**("커밋해줘"): 태그 선택 → 제목(**한글 25 / 영문 50자 이내**, 초과 시 다시) → 본문 → 메시지 `[태그] 제목`+빈 줄+본문으로 `git add -A && git commit`.
